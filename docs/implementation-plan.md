# 実装計画

最終更新: 2026-05-30

## 目的

GitHub の issue / commit / push 履歴を根拠として、メンバーのスキルを構造化し、Chat で使える説明文と参照情報を返せるようにする。

## 進め方

1. 履歴データを `github.md` から正規化して、メンバー・issue・commit の型に落とす。
2. issue の担当、ラベル、本文からスキルシグナルを抽出する。
3. commit メッセージ、変更ファイル、成果指標からスキルシグナルを抽出する。
4. メンバーごとのプロファイルを集約し、要約文とスコアを生成する。
5. Chat で使う文脈テンプレートを作成し、質問に応じて関連証拠を返す。
6. CLI を用意して、個人プロファイルとレポートを確認できるようにする。
7. 検証用テストを追加し、データ更新に強い形へ整える。
8. 最終的に、GitHub MCP 連携の差し替えがしやすい構造へ寄せる。

## 現在の判断

- データソースは `github.md` を基準に固定する。
- 本番外部連携はこの段階では入れず、履歴解釈の品質を優先する。
- スキル推定は決定論的なルールベースを基本にし、後から MCP や LLM を差し込める設計にする。
